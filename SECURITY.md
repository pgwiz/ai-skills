# Security Policy

Responsible disclosure

If you discover a security vulnerability in this repository or the Agent Memory Skill, please disclose it privately so it can be addressed before public disclosure.

1. Preferred method: Open a private GitHub Security Advisory for this repository. Include steps to reproduce, affected versions, and any proof-of-concept code.
2. If GitHub Security Advisories are not available, create a confidential issue and mark it as security, or contact the maintainers via the email address listed in the repository settings.

Response timeline

- Initial acknowledgment within 72 hours.
- Regular updates until a fix or mitigation is published.

What to include

- Detailed reproduction steps
- Minimal test case or proof-of-concept
- Impact assessment and suggested mitigation

DO NOT include sensitive data (API keys, secrets, or production credentials) in public issues.
